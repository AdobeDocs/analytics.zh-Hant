# 參與率

參與率度量會將成功事件的滿分指派給瀏覽期間傳遞的eVar所有值。 參與率量度對於判斷哪些頁面、促銷活動或其他自訂變數值對網站成功的貢獻最大，非常有用。 參與率是以瀏覽為基礎。 在事件發生之前及包含點擊時，瀏覽中的所有eVar值都會收到參與率評分，而不論過期設定為何。

請參閱[訪客參與率 - Ad Hoc Analysis](/help/components/c-variables/c-metrics/metrics-visitor-participation.md)，瞭解關於 Ad Hoc Analysis 如何使用參與率的詳細資訊。

參與率量度可對每個轉換事件進行兩項設定：

* **已停用**：每個轉換事件的預設狀態。不會為此事件收集參與率資料。
* **已啟用**：對此事件會收集參與率資料。

>[!NOTE] 您最多可對 100 個自訂事件啟用參與率。除此之外，您還可在[計算量度](https://marketing.adobe.com/resources/help/zh_TW/analytics/calcmetrics/participation_metric.html)產生器中建立參與率量度。

啟用後，參與率度量會自動在所有轉換報表中使用。 不過，參與率量度也可在您的要求下，在特定流量報表中檢視。 您可以選擇性地要求參與率量度可用於某些自訂流量報表。

## 收入參與率範例 {#section_DAB6C348201B454BB4ED01313AA777AF}

假設有下列序列：

1. 使用者導覽至您的網站並搜尋「鞋」。
1. 然後，使用者會搜尋「網球鞋」。
1. 使用者按一下產品頁面的連結，將項目新增至購物車，然後進行$120的購買。

在「內部搜尋詞」報告中顯示「收入」時，根據所選取的分配方法，您會看到下列項目：

* **第一個**：「鞋子」獲得 120 美元的評分。「網球鞋」可以獲得0美元。
* **最後一個**：「網球鞋」獲得 120 美元的評分。「鞋子」會得到0美元。
* **線性**：每個促銷活動各獲得 60 美元的評分。

   參與率類似於線性分配，但是所有值都會獲得滿分。 如果您使用收入（參與率）作為量度，則不考慮分配。 此範例中的收入（參與率）會針對兩個搜尋詞報告$120。
