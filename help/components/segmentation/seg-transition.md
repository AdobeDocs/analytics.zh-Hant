# 舊式區段

本文回答有關管理舊式區段最佳實務的常見問題。 舊式區段是建立於2014年之前的區段。

## 管理舊式區段 {#legacy}

+++ **我現有的區段有什麼改變？**

您現有的區段會如往常一樣繼續運作。 任何已套用這些區段的報表都會繼續正常運作。

大部分先前的預先定義和套裝區段會以區段範本的形式移轉至區段產生器。 區段範本可用以快速建立具有共同對象的自訂區段。 區段範本無法直接套用至報告，但可以輕易儲存成自訂區段。

區段範本在區段產生器中標示有特殊圖示![AdobeLogoSmall](/help/assets/icons/AdobeLogoSmall.svg)。



+++

+++ **已套用區段的已計劃報告有什麼改變？**

已計劃的報告會繼續使用您定義的區段正常運作。

當您刪除區段時，已套用此區段的已計劃報告和儀表板會繼續正常運作，亦即區段或儀表板會繼續使用刪除的區段。

當您編輯同名區段時，已計劃的報告不會更新。 以下是範例：假設您有 2 個同名區段分別位在不同報告套裝中：

![](assets/duplicate_seg_names.png)

您有個視覺效果是參照&#x200B;**[!UICONTROL mainprod]**&#x200B;報表套裝的區段。 之後您因為該區段重複而刪除該區段。 視覺效果會參照已刪除區段的定義繼續執行。 如果您將maindev區段的區段定義變更為包含Catalina Island和Tijuana （墨西哥），則套用至視覺效果的區段不會變更，而會使用舊定義。 若要使用新定義，請更新視覺效果以參考新定義。 如果您不確定視覺效果、專案或排程報表是否正使用已刪除的區段，請變更其餘區段的名稱，以顯示視覺效果是否正使用其餘的區段。

+++

+++ **Data Warehouse 區段有何變化？**

所有現有的 Data Warehouse 區段在 Data Warehouse 中仍可繼續運作。 大部分的Data Warehouse區段也可在Analysis Workspace等其他元件中運作。

您可以在區段產生器/管理員中建立或編輯新的Data Warehouse區段。 區段產生器中的產品相容性機制會自動判斷區段是否與Data Warehouse相容。

+++

+++ **預先建立的區段有何變化？**

* **單頁存取次數**
* **來自行動裝置的造訪次數**
* **來自免費搜尋的造訪次數**
* **來自付費搜尋的造訪次數**
* **具有訪客 ID Cookie 的造訪次數**

這些區段會以區段範本的形式移轉至區段產生器。 已套用這些區段的現有報告會繼續正常運作。

+++

+++ **CX Enterprise （套裝）區段有什麼改變：**

* 非購買者
* 購買者
* 首次造訪次數
* 來自社交網站的造訪次數
* 超過 10 分鐘的造訪次數*
* 5個以上先前造訪的造訪*
* 來自 Facebook 的造訪次數*

這些區段 (具有星號 * 標記者除外) 多半以區段範本的形式移轉至區段產生器。 另外，新增了數個新的區段範本。

已套用這些區段的現有報告會繼續正常運作。

+++

+++ **管理員區段 (也稱為「全域」區段有何變化？**

**管理員**&#x200B;區段已移轉至新的區段介面，並顯示為共用給每個人的區段。

這些區段的擁有者設定為擁有最舊管理員使用者帳戶的管理員。 但是，所有管理員都可以刪除、編輯和共用這些區段。

管理員在Admin Console中建立並管理這些全域區段的區段管理介面已無法使用。 管理員現在應使用新的區段產生器來建立區段，並與適當的群組、個人或所有人共用。

使用已如本檔案所述變更之邏輯的現有區段可繼續正常運作，不過區段必須先更新，才能再次儲存。 例如，如果您有現有的區段，其中&#x200B;**[!UICONTROL 美國州]** **[!UICONTROL 包含]** `New York`，該區段會繼續正常運作。 下次編輯區段時，您需要更新區段以使用具有&#x200B;**[!UICONTROL 等於]**&#x200B;條件的列舉型別。

+++

+++ **我應該如何處理具有相同名稱，但可能有不同定義的重複區段？** 
現在，由於區段可在多個報告套裝中運作，因此您可能發現有多個區段具有相同名稱。 您應該：

* 重新命名具有相同名稱但不同定義的區段，或
* 刪除不再需要的區段。

+++

+++ **Adobe對於清除區段有什麼建議？**

* 以舊版標記標記所有區段。
* 檢閱您擁有的區段。
* 將其新增至適用的區段程式庫。
* 核准規範區段。
* 根據[最佳實務](/help/components/segmentation/segmentation-workflow/seg-workflow.md)標記區段。

+++

### 移轉提示

下列提示可協助您移轉常見維度：

* 地域城市/地區/國家 - 搜尋並選取特定城市、地區或國家，而不是使用局部比對。
* 瀏覽器 — 使用瀏覽器型別維度來取得某個型別的所有瀏覽器，例如Google Chrome
* 作業系統 — 使用「作業系統型別」維度來取得某個型別的所有作業系統，例如Microsoft Windows。
* 請參閱「新維度和經過重新命名的維度」(參閱下文)

## 具有已知值的整數維度變更 {#integer-based-dims}

具有已知值集的整數維度（例如瀏覽器寬度）會分割為列舉範圍，以便您快速定義特定範圍的區段。 這些列舉清單會在維度名稱后面附加「 — 分段」。 下列畫面顯示使用舊的和新的「區段產生器」介面時，這些維度的分段方式：

![](assets/seg_browser_dimension.png)

小於、大於和類似的運運算元現在僅與Data Warehouse區段相容。 要與所有報表介面相容的區段，應使用含有&#x200B;**[!UICONTROL 等於]**&#x200B;運運算元的「分段」量度版本。
