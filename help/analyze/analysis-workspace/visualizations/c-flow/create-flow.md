# 設定流量視覺效果

>[!NOTE]
>
>此新版本 [!UICONTROL 流] 目前，可視化正在進行有限的測試。

更新後的流可視化功能使您能夠瞭解來自或通向網站或應用程式上特定轉換事件的過程。 它跟蹤通過維（和維項）或度量的路徑。 「流」(Flow)允許您配置您感興趣的路徑的開始或結束，或分析流經維或維項的所有路徑。

新 [!UICONTROL 流] 體驗可以通過多種方式增強工作流：

* 現在，您可以選擇使用度量和路徑維的組合來啟動或結束路徑。
* 它包含 [!UICONTROL 高級設定] 以便您進一步自定義 [!UICONTROL 流]。
* 新的「生成」按鈕允許您一次性配置行程，然後進行查詢，然後一次自動生成多個列和節點，從而節省分析時&#x200B;間。

![新建流UI](assets/new-flow.png)

## 配置步驟 {#configure}

1. 要開始建立流程圖，請向項目中添加一個空白麵板，然後按一下左滑軌中的可視化表徵圖。 然後將「流」可視化拖動到面板中。 或拖動 [!UICONTROL 流] 可視化到現有項目。

1. 使用以下三個選項之一鎖定流可視化：

   * [!UICONTROL 開始於] （度量、維或項），或
   * [!UICONTROL 包含] （尺寸或項），或
   * [!UICONTROL 結尾為] （度量、維或項）

   這些類別都會顯示在螢幕上的「拖放區域」中。可以通過三種方式填充放置區域：

   * 使用下拉菜單選擇度量或維。
   * 從維或度量清單中拖動項。
   * 使用搜索可查找要查找的度量或維。

   例如，假設您要跟蹤導致簽出事件的所有內容。 您將拖動與檢出相關的維或度量(如 [!UICONTROL 訂單存在]) **[!UICONTROL 結尾為]** 刪除區域。

1. 如果選擇度量，還需要提供 [!UICONTROL 路徑Dimension]，如圖所示，將用於生成路徑。 預設值為 [!UICONTROL 頁面]。

   ![路徑維](assets/pathing-dim.png)

   >[!IMPORTANT]
   >
   >無法將計算度量拖放到  **[!UICONTROL 開始於]** 或 **[!UICONTROL 結尾為]** 刪除區域。

1. （可選）按一下 **[!UICONTROL 顯示高級設定]** 配置高級設定：

   ![高級設定](assets/adv-settings.png)

   | 設定 | 說明 |
   | --- | --- |
   | **[!UICONTROL 包裝標籤]** | 一般而言，系統會截斷「流量」元素的標籤以節省螢幕空間，但您可勾選此方塊以完整顯示標籤。預設 = 未勾選。 |
   | **[!UICONTROL 包括重複實例]** | 「流量」視覺效果是根據維度的例項而定。此設定可讓您選擇要包含或排除重複例項，如頁面重新載入次數。不過，無法從包含多值維度 (例如 listVars、listProps、s.product、銷售 eVars 等) 的「流量」視覺效果中移除重複項目。預設 = 未勾選。 |
   | **[!UICONTROL 僅限於第一次/最後一次發生]** | 將路徑限制為以維/項/度量的第一次/最後一次出現開始/結束的路徑。 有關更詳細的說明，請參閱下面標題為「限制為第一次/最後一次出現的示例方案」的部分。 |
   | **[!UICONTROL 欄數]** | 確定要在流程圖中顯示的列數。 |
   | **[!UICONTROL 按列展開的項]** | 每列中需要的項目數。 |
   | **[!UICONTROL 流量容器]** | <ul><li>瀏覽</li><li>訪客</li></ul> 可讓您切換瀏覽和訪客，分析訪客路徑。這些設定可協助您了解訪客層級的訪客參與程度 (跨越造訪)，或是將分析限制在單一造訪。 |

1. 按一下&#x200B;**[!UICONTROL 「建置」]**。

## 查看和更改流輸出 {#output}

![流量輸出](assets/flow-output.png)

「流」配置的摘要顯示在圖的頂部。 圖表中的路徑會依照比例顯示。具有較多活動的路徑看起來比較寬。

要進一步深入查看資料，您有以下幾個選項：

* 流量圖是互動式的。將滑鼠移到圖表上方，可變更顯示的詳細資料。

* 按一下圖表中的節點，會顯示該節點的詳細資料。再次按一下該節點以折疊它。

   ![節點詳細資訊](assets/node-details.png)

* 您可以篩選列以僅顯示某些結果，如包括和排除、指定條件等。

* 按一下左邊的加號(+)展開列。

* 使用下面介紹的按一下右鍵選項進一步定制輸出。

* 按一下配置摘要旁邊的鉛筆表徵圖，以進一步編輯流或使用不同的選項重建流。

* 您也可以前往&#x200B;**[!UICONTROL 「專案]** > **[!UICONTROL 下載 CSV」]**，匯出並進一步分析您專案 .CSV 檔案的「流量」圖表。

## 篩選

在每列上方懸停時，將顯示一個篩選器。 通過按一下篩選器，可獲得當前存在於「自由形式」(Freeform)表格中的相同篩選器對話框。 此篩選器與在「自由形式」(Freeform)表格中相同。

* 使用高級設定將某些條件包含或排除在運算子清單中。
* 從清單中篩選項目後，該特定列將反映篩選。 (篩選器會將其縮減為僅顯示篩選器中允許的項，或刪除除篩選器中要包含的一個項以外的所有項。
* 所有下游列和上游列應保持，只要有資料流入其餘節點。
* 應用後，篩選器表徵圖將顯示在正在篩選的列上方的藍色中。
* 要刪除篩選器，請按一下篩選器表徵圖以開啟篩選器菜單。 刪除所應用的任何篩選器，然後按一下 **[!UICONTROL 保存]**。 流應返回其以前未篩選的狀態。

| [!UICONTROL 自這個流量的點建立區段] | 建立區段。此選項會將您帶往「區段產生器」，讓您在該處設定新區段。 |
| [!UICONTROL 劃分] | 依據可用的「維度」、「量度」或「時間」來劃分節點。 |
| [!UICONTROL 趨勢] | 建立節點的趨勢圖。 |
| [!UICONTROL 展開整個欄] | 展開該欄以顯示所有節點。依預設，只會顯示前五個節點。 |

## 「限制為第一次/最後一次出現」的示例方案

使用此選項時，請記住：

* **[!UICONTROL 限制為第一次/最後一次出現]** 只計算系列中的第一個/最後一個事件。 所有其他出現的 **[!UICONTROL 開始於]** 或 **[!UICONTROL 結尾為]** 將放棄條件。
* 如果與 **[!UICONTROL 開始於]** 流，只包括與開始條件匹配的第一個實例。
* 如果與 **[!UICONTROL 結尾為]** 流，將只包括與結束條件匹配的最後一次出現。
* 使用的系列因容器而異。 如果使用 **[!UICONTROL 訪問]** 容器，一連串命中將是會話。 如果使用 **[!UICONTROL 訪問者]** 容器，該系列命中將是給定用戶在提供日期範圍內的所有命中。
* 的 **[!UICONTROL 限制為第一次/最後一次出現]** 在「開始於」或「結束於」欄位中使用度量或Dimension項時，可以在高級設定中配置選項。

命中數系列示例：

首頁>產品>添加到購物車>產品>添加到購物車>開單>訂單確認

### 使用以下設定考慮流分析：

* 開始於[!UICONTROL  添加到購物車] (Dimension項)
* [!UICONTROL 頁面] 路徑維
* [!UICONTROL 訪問] 容器

如果禁用「限制為第一次/最後一次出現」，則此單一系列命中將計數2次「添加到購物車」。
預期流輸出：&quot;添加到購物車&quot;(2)—> &quot;產品&quot;(1)-> &quot;計費&quot;(1)

但是，如果啟用「限制為第一次/最後一次出現」，則分析中將只包含「添加到購物車」的第一次出現。
預期流輸出：&quot;添加到購物車&quot;(1)—> &quot;產品&quot;(1)

### 請考慮相同的命中序列，但使用以下設定：

* 結尾為 [!UICONTROL 添加到購物車] (Dimension項)
* [!UICONTROL 頁面] 路徑維
* [!UICONTROL 訪問] 容器

如果 **[!UICONTROL 限制為第一次/最後一次出現]** 是 *禁用*，則此單一系列命中次數將計數2次「添加到購物車」。
預期流輸出：&quot;產品&quot;(2)&lt;— &quot;添加到購物車&quot;(2)

但是，如果 **[!UICONTROL 限制為第一次/最後一次出現]** 是 *啟用*，僅上次出現 [!UICONTROL 添加到購物車] 會被包括在分析中。
預期流輸出：&quot;產品&quot;(1)&lt;— &quot;添加到購物車&quot;(1)
