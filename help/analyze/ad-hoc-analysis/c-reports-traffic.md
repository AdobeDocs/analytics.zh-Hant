# 流量來源報表

流量來源報表能讓您深入了解訪客與您網站的互動。

## 流量來源報表 {#concept_0F1772141E1345C5BCF63BE7C544C0CB}

流量來源報表能讓您深入了解訪客與您網站的互動。

流量來源報表可讓您：

* 分析訪客行為的重要方面。
* 監視和了解流量模式。
* 決定熱門網站內容。
* 依任何可衡量的標準來劃分訪客。

**通用永續性**

在中 [!UICONTROL Traffic Sources]，所有報表值都會持續存在並接收評分，直到被覆寫或瀏覽結束（以先到者為準）為止。 以前，只有關鍵字和反向連結網域會持續存在。 例如，如果訪客執行      「DVD」會將他們帶到您的網站進行$100的購買，報表會將$100評分分配給關鍵字「DVD」以及Google搜尋引擎。 不論 [!DNL Admin Console] 設定為何，該功能皆不可變更。

## 搜尋關鍵字 {#concept_071FDCBD0A3B4242BA00744786D1C59C}

顯示「全部」、「付費」和「免費」搜尋的關鍵字劃分。

<!-- 

c_reports_search_keyword.xml

 -->

**[!UICONTROL Search Keywords - All]**:顯示用於尋找您網站的每個搜尋關鍵字的劃分資訊。 您可以按一下清單上方的欄標題，依頁面檢視或搜尋關鍵字來排序此清單。 按一下搜尋關鍵字旁的放大鏡，查看您網站的搜尋結果。

**[!UICONTROL Search Keywords - Paid]**:顯示用於尋找您網站的每個付費搜尋關鍵字的劃分資訊。 您可以按一下清單上方的欄標題，依頁面檢視或搜尋關鍵字來排序此清單。 按一下搜尋關鍵字旁的放大鏡，查看您網站的搜尋結果。

**[!UICONTROL Search Keywords - Natural]**:顯示用於尋找您網站的每個免費搜尋關鍵字的劃分資訊。 您可以按一下清單上方的欄標題，依頁面檢視或搜尋關鍵字來排序此清單。 按一下搜尋關鍵字旁的放大鏡，查看您網站的搜尋結果。

## 搜尋引擎 {#concept_351CDE4F5FC44371B6B657064E125134}

顯示訪客用來進行所有、付費、免費搜尋的搜尋引擎。

<!-- 

c_reports_search_engines.xml

 -->

**[!UICONTROL Search Engines - All]**:顯示訪客使用哪些搜尋引擎找到您的網頁。 圖表顯示用於尋找您網站的搜尋引擎的百分比劃分。

**[!UICONTROL Search Engines - Paid]**:顯示訪客使用哪些付費關鍵字搜尋引擎找到您的網頁。 圖表顯示用於尋找您網站的搜尋引擎的百分比劃分。

**[!UICONTROL Search Engines - Natural]**:顯示訪客使用哪些免費關鍵字搜尋引擎找到您的網頁。 圖表顯示用於尋找您網站的搜尋引擎的百分比劃分。

## 反向連結網域 {#concept_804614DF21C14C9FB542451B30F92788}

<!-- 

c_reports_ref_domains.xml

 -->

顯示對網站的「成功量度」影響最大之客戶的反向連結網域。反向連結分為 2 種主要類別：網域與 URL。網域會參照網域名稱，並顯示為基本網域，而不會附加查詢字串或子目錄。 URL 包括基本網域名稱，亦包括任意查詢字串或子目錄。

## 原始反向連結網域 {#concept_EB18251DF70343169B46BB59543A579A}

<!-- 

c_reports_original_ref_domains.xml

 -->

顯示在您的網站上產生客戶的原始反向連結。 客戶可以多次瀏覽您的網站，每次瀏覽都有不同的反向連結。 此報表顯示客戶首次造訪您網站時的反向連結。 您可以透過該等資訊了解是否客戶繼續使用原始反向連結和檢視模式。您可以檢視原始反向連結產生的訪客數量，或瞭解每個原始反向連結產生的收入。 可在訪客每次存取您網站時填入反向連結報告，即使此訪客在一個工作階段中 (瀏覽過期之前) 多次存取此網站。

## 反向連結 {#concept_40CF9C2D10B94E82819BC65A232F05C3}

顯示訪客在到達您的網站前所處的網域或URL、訪客尋找您網站時所使用的方法，以及這些反向連結位置所造成的瀏覽次數。

<!-- 

c_reports_referrers.xml

 -->

例如，如果訪客從網站A點按連結並到達您的網站，則網站A若未定義為您網域的一部分，即為反向連結。 在行銷報告與分析實作期間，您的實施顧問可協助您定義屬於您網站一部分的網域和URL。 （此變更可在實施後完成）。

不屬於這些已定義網域和URL的網域或URL，則視為反向連結。 例如，網頁A和網頁B會新增至內部URL篩選器，但網頁C則否。 在此例中，網頁C被視為反向連結。

請參閱 [!DNL Admin Console] 說明中的[內部 URL 篩選器](https://marketing.adobe.com/resources/help/zh_TW/reference/internal_URL_filter_admin.html)，了解詳細資訊。

>[!NOTE] 訪客點選內含通訊協定 [!DNL imap://] 或 [!DNL mail://] 的電子郵件連結而進入您的網站時，Marketing Reports &amp; Analytics 會將反向連結網域記錄為電子郵件。例如，來自 [!DNL https://mail.yahoo.com] 的任何點按不會被計算為電子郵件反向連結，因為通訊協定為 [!DNL https://]。來自 Outlook 的電子郵件會報告到「分類/建立書籤」明細項目，而任何已知搜尋引擎網域處的 HTTP 通訊協定反向連結會報告到「搜尋引擎」明細項目。

## 反向連結類型 {#concept_689E42D8F96C450DA41C7167C7388198}

透過追蹤並記錄每次存取的訪客反向連結位置，您可以了解每次存取中訪客是如何找到您的網站的。

<!-- 

c_reports_ref_types.xml

 -->

下列清單定義反向連結的各種類型：

* *當訪客點按* 其他網站（未定義為您網站的一部分）頁面上的連結並進入您的網站時，會記錄其他網站反向連結。
* *當訪客使用* 搜尋引擎存取您的網站時，會記錄搜尋引擎反向連結。
* 以下情形會記錄&#x200B;*分類/建立書籤*&#x200B;反向連結：

   * 如果訪客透過非瀏覽器連結進入您的網站（例如，在電子郵件中）。
   * 如果訪客在瀏覽器中直接輸入您網站的 URL。
   * 如果訪客點按其個人硬碟上的HTML連結。
   * 如果訪客透過選擇瀏覽器書籤存取您的網站。

**定義**

執行此報表時可能會顯示下列明細項目：

**網站內**：這些項目是指被內部 URL 篩選器加上標記的 URL。這些項目不會計為反向連結例項，但可在報告其他度量時顯示。

**無 JavaScript**：沒有 JavaScript，因此無法識別類型 (未知)。這表示瀏覽器上的用戶端未提供反向連結資訊，因此無法報告可支援Javascript。 這些不會計為「反向連結例項」，但可在報告其他度量時看到。

**USENET (新聞群組)**：這表示反向連結的 URL 是以 `news://` 開頭的。因此，反向連結被發佈在 Usenet 新聞群組上，而不是網頁上。

>[!NOTE] 「反向連結類型」邏輯與其他流量來源報表( [!UICONTROL Referrers] 如 [!UICONTROL Referring Domains]和)相符。 This should reduce or eliminate the occurrences of the Inside Your Site and No JavaScript line items in the [!UICONTROL Referrer Type] report.
