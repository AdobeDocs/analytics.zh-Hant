# Adobe Analytics 中的低流量值

當報表中有大量不重複值時，Adobe 提供的功能可確保最重要的值出現在您的報表中。在大約500,000個現有值之後收集的唯一變數值列在標有的維度項下 **[!UICONTROL 低流量]**。

## [!UICONTROL 低流量]的運作方式

* 如果指定月份中變數未達到 500,000 個不重複值，則不會影響報表。
* 當變數達到500,000個唯一值時，資料開始分時段 [!UICONTROL 低流量]。 超過此臨界值的每個值都會依照下列邏輯執行後續作業：
   * 如果值已存在於報表中，則照常新增該值。
   * 如果報表中尚未看到值，則它最初在 [!UICONTROL 低流量] 維項。
   * 如果分時段在 [!UICONTROL 低流量] 接收大量流量（通常在一天內以兩位數表示的實例），它開始被識別為自己的維項。 在達到閾值之前收集的實例仍低於 [!UICONTROL 低流量]。 確切閾值具有許多相關性，如處理報表套件資料的伺服器數量和每個維項目實例之間的時間量。
* 如果報告套件的唯一值超過1,000,000個，則應用更加激進的篩選。 唯一值要求實例在被識別為其自己的維度項之前，在一天內以三位數表示。

此邏輯允許Adobe優化報告功能，同時仍允許您的組織報告本月晚些時候收集的關鍵維項。 例如，您的組織運行的站點包含數百萬篇文章，而且新文章在接近月底時（在超過兩個唯一閾值後）變得流行。 您仍然可以分析該文章的效能，而不必在其下 [!UICONTROL 低流量]。 此邏輯不是要取消儲存每天或每月獲得一定數量頁面視圖的所有內容。

>[!NOTE]
>的 [頁面](../components/dimensions/page.md) 維使用多個後端列，所有列都計入唯一閾值，包括 `pagename`。 `page_url`。 `first_hit_pagename`。 `first_hit_page_url`。 `visit_pagename`。 `visit_page_url`, `click_context`。 這些後端列可能導致 [!UICONTROL 低流量] 在工作區中唯一的頁維項數達到500,000之前應用的邏輯。

## 變更不重複限制臨界值

這些臨界值限制預設為 500,000 和 100 萬個唯一值。這些限制可依各變數變更。聯繫Adobe客戶服務或您組織的客戶經理以請求此更改。 要求變更時，請提供以下資料：

* 報告套裝 ID
* 您想要提高臨界值的變數
* 需要的第一臨界值和第二臨界值

變更臨界值可能會影響報表效能。Adobe 強烈建議，要求增加某變數中的不重複值時，務必做出正確的判斷。僅增加對組織報告需求至關重要的變數的唯一限制。

Analytics 使用者介面中不會顯示低流量臨界值。如果希望獲得有關現有閾值的詳細資訊，請與Adobe客戶服務部門聯繫。

## 使用元件和其他功能的低流量

不同的功能可能會以不同的方式處理低流量值。

* **Data Warehouse：** Data Warehouse 報表中的不重複值數目沒有限制。其獨特的架構可將任意數目的不重複值納入報表中。
   * 在某些有限的情況下，低流量值仍可能顯示。範例包括清單變數、清單 prop、銷售 eVar 和行銷管道詳細資訊維度。
* **分段：**&#x200B;如果區段準則包含具有大量不重複值的變數，則不會包含低流量底下擷取的值。
* **分類：**&#x200B;分類報表也受到不重複限制的約束。如果分類的上層變數值包含在低流量底下，則不會分類該值。
   * 若是透過匯入工具取得的低流量分類值，您可在 Data Warehouse 中查看。<!-- AN-115871 -->
   * 若是透過規則產生器取得的低流量分類值，*無法*&#x200B;在 Data Warehouse 中查看。<!-- AN-122872 -->
