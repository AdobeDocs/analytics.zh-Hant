# 伺服器呼叫使用量常見問答

| 問題 | 回答 |
|--- |--- |
| 問：我是 Analytics 管理員。 為什麼我會在登入公司的「管理員」區段底下看到「[!UICONTROL 伺服器呼叫使用量]」連結？ | 答：您可以選擇哪些登入公司會在「管理員」區段底下顯示「[!UICONTROL 伺服器呼叫使用量]」連結。 如果沒有看見，請聯絡客戶服務以將其啟用。 |
| 問：「[!UICONTROL 伺服器呼叫使用量]」是否為僅限管理員使用的功能？ | 答：否。 就像「[!UICONTROL 伺服器呼叫使用量]」所取代的「[!UICONTROL 計費]」功能一樣，您可以指派權限給非管理員以存取此連結。 |
| 問：「[!UICONTROL 伺服器呼叫使用量]」和承諾使用量的詳細資料是否專屬於單一登入公司？ | 答：否。 所顯示的使用情況和承諾使用詳細資訊涵蓋您受Adobe Analytics合約管轄的所有登入公司。 |
| 問：目前登入的公司有些報表套裝無法提供報表資料，但是我在「[!UICONTROL 報表套裝使用量]」索引標籤下仍會看到這些報表套裝的耗用量。 這是否為錯誤？ | 答：否。 就像您的「[!UICONTROL 伺服器呼叫使用量]」和承諾使用量詳細資料一樣，報表套裝清單涵蓋您根據 Analytics 合約實作的所有登入公司。 |
| 問：使用期間是否一律與我的 Analytics 合約或計費週期的期間相同？ | 答：否。 使用期間是指您的伺服器呼叫承諾付款適用的期間，且可能與您的合約時間表或計費週期不同。 例如，您的年度合約可讓您每月有10億次伺服器呼叫，在這種情況下，您的承諾使用量為10億次伺服器呼叫，而您的使用期為一個月。 |
| 問：我的合約包括視訊心率的承諾使用量，但我在「[!UICONTROL 伺服器呼叫使用量]」底下沒有看見。 我是否應該聯絡客戶服務？ | 答：視訊心率的耗用量和承諾使用量詳細資料並不屬於「[!UICONTROL 伺服器呼叫使用量]」。 |
| 問：我的合約並未包含次要伺服器呼叫的佈建，但我仍可在「[!UICONTROL 目前使用量]」和「[!UICONTROL 報表套裝使用量]」索引標籤底下看到這些呼叫。 這是否為錯誤？ | 答：否。 雖然您的合約並未明確布建次要伺服器呼叫，但如果您將Analytics資料傳送至多個報表套裝，還是可以產生這些呼叫。 您可以重新造訪您的實作，以確保您沒有傳送此類呼叫，或聯絡您的Adobe客戶團隊以更新您的合約，使其包含此類呼叫的布建。 |
| 問：我的合約並未包含次要伺服器呼叫的佈建，但我仍可在「[!UICONTROL 目前使用量]」和「[!UICONTROL 報表套裝使用量]」索引標籤底下看到這些呼叫。 這是否表示我開始發生使用過量的狀況？ | 答：您在未佈建的情況下，傳入次要伺服器呼叫時，是否開始發生使用過量情形，這一點取決於您的合約。在某些情況下，次要伺服器呼叫耗用量可能會以您對主要伺服器呼叫的承諾使用量為計算基準，因此消耗速度會比您預期的更快。 在其他情況下，即使您尚未使用完所有主要伺服器呼叫，也可能需要為次要伺服器呼叫付費。請參閱您的合約，或聯絡您的Adobe客戶團隊以進行確認。 |
| 問：我的合約/使用期間才剛開始，卻已經收到關於超出次要伺服器呼叫承諾使用量的警報。 這是否為錯誤？會開始向我收取使用過量的費用嗎？ | 答：否。 這表示您會使用次要伺服器呼叫，而不會將其明確布建為Adobe Analytics合約的一部分。 此警示只是為了確保您知道此情況，並讓您在需要時採取修正動作。 另請參閱上述的常見問答 8。 |
| 問：我未建立任何「[!UICONTROL 伺服器呼叫使用量]」警報，但我收到多個這類警報，且所有警報都是相同的內容。 這是否為錯誤？ | 答：否。 在「[!UICONTROL 管理]」區段底下啟用「[!UICONTROL 伺服器呼叫使用量]」連結的所有登入公司都會觸發預設警報。 警報包括觸發該警報的公司名稱。 身為管理員，您可以停用及/或刪除這些警報以避免重複、從收件者清單中刪除自己，或讓客戶服務停用特定公司的「[!UICONTROL 伺服器呼叫使用量]」連結。 |
